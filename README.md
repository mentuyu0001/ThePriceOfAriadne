## 技術的なこだわり

本作では「直感的な操作性」と「安定したパフォーマンス」の両立を掲げ、以下の技術的な工夫を行いました。

### 1. 「手触り」を追求したプレイヤー制御
入力と画面反応のズレによるストレス（理不尽さ）を排除するため、物理挙動に対して独自の調整を加えています。

* **動的な加速度制御**: 現在の速度に応じて加速度を動的に変化させ、初動の重さと走り出しの爽快感を両立する「緩急」のある挙動を実装。
* **判定の緩和**: コライダーが厳密に接地していなくても、視覚的に接地しているように見えればジャンプ可能とする判定調整。
* **コヨーテタイム (Coyote Time) の導入**: 足場から踏み外した直後の数フレーム間もジャンプ入力を受け付ける猶予時間を実装し、ギリギリのアクションを許容する快適な操作感を実現。

### 2. 並列処理とロード最適化によるパフォーマンス維持
多数のオブジェクトやギミックが存在するマップでもフレームレートを落とさないよう、最適化を行いました。

* **UniTaskによる非同期処理**: 計算負荷の高い処理をメインスレッドから切り離し、並列化することでスパイク（カクつき）を防止。
* **事前ロード機構**: シーン遷移時のブラックアウト時間を解消するため、バックグラウンドでの事前ロードシステムを構築。
* **コードのリファクタリング**: 重複処理の共通化や、Update内での高負荷処理の排除を徹底し、保守性とパフォーマンスを向上。

### 3. プランナー主導の調整を可能にする設計
ゲームバランスの調整効率を最大化するため、パラメータ管理に `ScriptableObject` を採用しました。
移動速度、ジャンプ力、判定範囲などの変数をコードから分離し、インスペクター上で一元管理できるように設計。これにより、プログラマーの手を介さずにプランナーがパラメータを微調整し、即座にプレイテストを行える高速なイテレーション環境を構築しました。
